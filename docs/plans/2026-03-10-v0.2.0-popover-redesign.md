# v0.2.0 — Popover Redesign + Ring Metric Toggle

Date: 2026-03-10

## Summary

Two features for v0.2.0: a polished popover UI replacing the default NSPopover styling, and a toggle that lets users choose whether the menu bar ring shows session or weekly usage.

## Feature 1: Ring Metric Toggle

### What it does

A segmented control ("Session" | "Weekly") in the popover controls what the menu bar ring displays.

- **Session mode:** ring shows `currentSession.percentUsed`
- **Weekly mode:** ring shows `max(weeklyAllModels.percentUsed, weeklySonnetOnly.percentUsed)` (current behavior)

### Persistence

- Stored in `UserDefaults` under key `ringMetricMode` (`"session"` or `"weekly"`)
- Default: `"session"`
- Read on launch, updated when user taps the toggle

### Ring color logic

Unchanged. `RingColorState.from(percent:)` still applies to whatever value the ring is showing.

## Feature 2: Polished Popover UI

### Visual style

- **Option B from brainstorm:** clean white card (light mode), dark card (dark mode)
- Follows system appearance via AppKit semantic colors (`NSColor.windowBackgroundColor`, `.labelColor`, `.secondaryLabelColor`, `.separatorColor`, etc.)
- Rounded corners (10pt radius on content view)
- Custom thin progress bars (4pt height, 2pt corner radius) replacing `NSProgressIndicator`
- Proper typography: 13pt semibold title, 11pt medium labels, 11pt monospaced-digit percentages, 10pt tertiary reset times

### Refresh button

Custom SVG-based icon using teenyicons refresh (MIT license). The SVG path is drawn via `NSBezierPath` at render time — no image assets needed. Strokes with `NSColor.secondaryLabelColor` so it adapts to light/dark automatically.

### Progress bar colors

| Usage % | Bar color |
|---------|-----------|
| 0–59%   | `.labelColor` (black in light, white in dark) |
| 60–84%  | Amber `#F59E0B` |
| 85–100% | Red `#EF4444` |

Same thresholds as the ring, applied per-row independently.

## Feature 3: Auth-Aware Footer

- `MenuViewController` exposes an `updateAuthState(isSignedIn: Bool)` method
- When signed in: footer shows "Sign Out" + "Quit"
- When signed out: footer shows "Sign In" + "Quit", usage rows hidden, empty state shown
- `AppDelegate` calls `updateAuthState` in response to `AuthManagerDelegate` callbacks

## Architecture changes

### Files modified

- `Models.swift` — add `RingMetricMode` enum
- `MenuView.swift` — full rewrite of `MenuViewController` with custom drawing, segmented control, auth-aware footer
- `AppDelegate.swift` — wire up ring metric preference, pass auth state to menu view, add `onSignIn` callback
- `UsageData.ringValue` — make it take a `RingMetricMode` parameter instead of always using max

### Files added

- `RefreshIcon.swift` — `NSView` subclass that draws the teenyicons refresh SVG via `NSBezierPath`

### No changes to

- `UsageService.swift` — still fetches all three metrics regardless of toggle
- `RingView.swift` — still draws whatever progress value it's given
- `AuthManager.swift` — existing sign-in/sign-out logic unchanged
- `LoginWindowController.swift` — unchanged

## Dark mode strategy

Use AppKit semantic colors throughout. No hardcoded hex values except for amber (`#F59E0B`) and red (`#EF4444`) which are the same in both modes. The popover's `NSVisualEffectView` or `wantsLayer` + semantic background handles the rest automatically.

Key color mappings:
- Card background: `.windowBackgroundColor`
- Title text: `.labelColor`
- Label text: `.secondaryLabelColor`
- Sublabel text: `.tertiaryLabelColor`
- Progress track: `.separatorColor`
- Progress fill (normal): `.labelColor`
- Separators: `.separatorColor`
- Segmented control background: `.controlBackgroundColor`
- Segmented control selected: `.selectedContentBackgroundColor` or `.controlColor`
